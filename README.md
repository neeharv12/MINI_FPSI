# MINI-FPSI

This repository contains the following artifacts:

1. Prototype implementation of the MINI-FPSI protocol used in the paper, including C++ source code and a CMake build file.
2. Reproducibility scripts for benchmarking, as used in Table 2 of the paper.
3. WAN emulation scripts.
4. Dependency build instructions.

## Requirements

- A C++20 compiler.
- CMake 3.15 or newer.
- [libOTe](https://github.com/osu-crypto/libOTe), built with Boost and libsodium support.
- [toxiproxy](https://github.com/Shopify/toxiproxy), for WAN benchmarks.

## Building libOTe

libOTe is included in this repository under `./libOTe/`. Build it, then install it to a project-local prefix:

```bash
cd libOTe
python build.py --all --boost --sodium
python build.py --install=$(pwd)/../libOTe-install
cd ..
```

## Building this project

From the project root:

```bash
cmake -S . -B build \
  -DCMAKE_PREFIX_PATH=$(pwd)/libOTe-install \
  -DENABLE_BOOST=ON \
  -DENABLE_SODIUM=ON \
  -DCOPROTO_ENABLE_BOOST=ON
cmake --build build -j 2
```

This produces two binaries: `build/Alice` and `build/Bob`.

## Running

Alice and Bob run as separate processes that connect over a TCP socket. **Start Bob first** so he's listening when Alice connects.

In one terminal:

```bash
./build/Bob \
  --addr 127.0.0.1:1212 \
  --b inputs/B.txt \
  --ot iknp \
  --l 32 \
  --t 16 \
  --threads 16
```

In a second terminal:

```bash
./build/Alice \
  --addr 127.0.0.1:1212 \
  --a inputs/A.txt \
  --ot iknp \
  --l 32 \
  --t 16 \
  --threads 16
```

## Command-line options

| Flag | Default | Applies to | Description |
|---|---|---|---|
| `--addr ip:port` | `127.0.0.1:1212` | both | TCP address. Bob binds, Alice connects. |
| `--threads N` | `2` | both | Worker threads for parallel-for and the Asio runtime. |
| `--seed S` | `0` | both | PRNG seed. `0` means use system randomness. |
| `--verbose` | off | both | Per-pair debug output. |
| `--ot iknp\|silent\|softspoken` | `iknp` | both | OT extension backend. Both sides must agree. |
| `--l L` | `32` | both | Bitstring length. Must satisfy `l + 1 <= 128`. |
| `--t T` | `16` | both | Hamming-distance threshold. Must satisfy `t <= l`. |
| `--k K` | `6` | both | SoftSpoken tunable parameter. Ignored by other schemes. |
| `--a path` | — | Alice only | Path to Alice's input file. Required. |
| `--b path` | — | Bob only | Path to Bob's input file. Required. |

Alice and Bob must be invoked with matching `--ot`, `--l`, `--t`, and `--k` values.

## Input file format

Both `A.txt` and `B.txt` are plain text, one bitstring per line, every line exactly `l` characters of `0` or `1`. Empty lines are skipped; any other character is rejected. The benchmarking scripts generate `A.txt` and `B.txt` randomly. For independent runs, supply `A.txt` and `B.txt` files whose line length matches the `--l` parameter.

## localhost benchmarks

`test.py` sweeps `(scheme, n, l, t, k)` and writes one log per configuration into `the_final_output/`. It generates fresh random `A.txt` / `B.txt` for each run, starts Bob, waits half a second for the socket to bind, then runs Alice, and captures both processes' output.

```bash
python3 test.py
```


## WAN benchmarks

For benchmarking over simulated WAN, install [toxiproxy](https://github.com/Shopify/toxiproxy) and use `wan.sh` to start it.

Install toxiproxy.

On Linux:

```bash
wget https://github.com/Shopify/toxiproxy/releases/download/v2.12.0/toxiproxy_2.12.0_linux_amd64.deb
sudo dpkg -i toxiproxy_2.12.0_linux_amd64.deb
```

On macOS:

```bash
brew install toxiproxy
```

Then make `wan.sh` executable and run the sweep:

```bash
chmod +x wan.sh
./wan.sh start
python3 test_wan.py
./wan.sh stop
```

`./wan.sh start` defaults to 70 ms RTT and 100 Mbit/s. To override: `./wan.sh start <one_way_latency_ms> <bandwidth_kbps>`.
